Key risk — Legal jurisdiction: Alibaba Group is headquartered in China. Chinese law (National Intelligence Law 2017) can compel data disclosure to government authorities. 6.3  Self-Managed Capabilities — What Your Team Controls

Qoder’s extensible architecture lets teams manage their own MCP servers, Skills, and Custom Agents. These stay within your control — no platform dependency.

Your Team Controls

MCP Servers

  • Connect to any MCP-compatible service
  • Build custom servers for internal APIs, DBs, docs
  • 3 types: Streamable HTTP, SSE, STDIO (local CLI)
  • Config per-project or per-user
  • MCP server runs on your infra — acts as a controlled gateway

Docs: MCP Configuration

Your Team Controls

Skills

  • Markdown files (SKILL.md) with instructions
  • Project-level: .qoder/skills/ → Git-versioned, team-shared
  • User-level: ~/.qoder/skills/ → personal
  • Can include scripts, templates, reference docs
  • Encode coding standards, security checklists, compliance

Docs: Skills

Your Team Controls

Custom Agents

  • Specialized AI agents with own context + permissions
  • Restrict per agent: tools, MCP servers, skills, models
  • Project-level: .qoder/agents/ → team-shared via Git
  • Examples: read-only reviewer, security auditor, deploy agent
  • Principle of least privilege built-in

Docs: Custom Agents

Team Admin (Enterprise)

Admin Controls

  • SSO / SAML single sign-on
  • RBAC — role-based permission management
  • Domain-based access restriction
  • Privacy mode enforcement for all members
  • Centralized billing + usage analytics
  • Gap: No documented controls to restrict which MCP/skills devs use

Docs: Teams Setup · Enterprise

Security positive: MCP, Skills, and Custom Agents are all stored locally or in Git. Teams control their own extensions — internal tools, coding standards, and workflows never leave the organization. Custom MCP servers act as controlled gateways to internal systems, enforcing auth and filtering before data reaches the model.
Governance gap: No documented admin control to restrict which MCP servers, skills, or agents developers can use (e.g., preventing a developer from connecting an unauthorized MCP server to a production database). Discuss with Qoder Enterprise.
