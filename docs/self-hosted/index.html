Why Self-Host?

A 50-person team currently on Opus 4.6 + Claude Code — is self-hosting worth it? The table below compares the two options across the dimensions that matter for daily coding and analysis.

Opus 4.6 + Claude CodeSelf-Hosted (AWS)
Data privacyPrompts transit provider infraAll data stays in your VPC
Cost (50 devs)$30–60K/mo (Opus 4.6 pricing)~$12K/mo reserved instances; break-even ~2 mo
ModelsOpus 4.6 (proprietary, frontier)DeepSeek, Qwen, GLM, Kimi (open-weight)
Quality gapFrontier baseline~5% below on daily coding; ~15–25% below on complex agentic tasks (multi-file refactors, architecture)
Harness effectClaude Code: proprietary agent harness (tool use, planning, context mgmt) tightly coupled to Opus 4.6OpenCode: open-source harness — same model scores differently; harness gap compounds model gap
Ops burdenNone1–2 engineers for EKS/GPU platform
LatencySubject to provider loadTunable; TTFT p99 <5s achievable
Self-hosting makes sense if data sovereignty is non-negotiable or Opus 4.6 spend exceeds $30K/mo. The quality gap is real: ~5% on daily coding but 15–25% on complex agentic tasks, compounded by the harness gap (Claude Code vs OpenCode). Open-weight wins on latency, fine-tuning, and unlimited context. Consider a hybrid: self-host for sensitive repos and high-volume daily coding, keep Opus 4.6 + Claude Code for complex architecture and multi-step reasoning where the frontier edge matters.
