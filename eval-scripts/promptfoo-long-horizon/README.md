# Promptfoo Long-Horizon Implementation

This folder implements the **Long-Horizon** SDLC evaluation described in the self-hosted LLM evaluation page. It assumes candidate artifacts have already been generated by the replacement model and harness.

The intended flow is:

```text
Existing SDLC case
  -> candidate artifact generated by replacement model + harness
  -> Promptfoo echo provider passes the artifact to assertions
  -> hard checks validate required structure
  -> LLM-as-Judge scores the four core metrics
  -> exported report is summarized into a weighted Long-Horizon score
```

## Why Echo Provider

The SDLC artifacts are already generated by the candidate model and harness before evaluation. Promptfoo's `echo` provider returns the candidate artifact as the model output, so Promptfoo can judge pre-generated artifacts without calling the candidate model again.

## Files

```text
promptfooconfig.yaml          Main LLM-as-Judge config
promptfooconfig.smoke.yaml    Dependency-light smoke config
score_model.json              Core metric weights and purposes
case_schema.json              Required fields for each SDLC case
prompts/candidate_artifact.txt
tests/sdlc_cases.yaml         Example SDLC cases and candidate artifacts
scripts/summarize_results.py  Small helper for exported Promptfoo JSON
```

## Case Preparation

Each Promptfoo test case represents one generated SDLC artifact for one stage.

Required fields:

- `case_id`: stable project or case identifier
- `stage`: SDLC stage being evaluated
- `required_id`: identifier that must be preserved
- `requirement_context`: original user brief or requirement context
- `prior_artifact`: previous-stage artifact
- `claude_baseline`: accepted production Claude artifact
- `candidate_artifact`: artifact generated by the replacement model and harness

Keep candidate generation outside Promptfoo. Promptfoo evaluates the generated artifact; it does not run the candidate model in this implementation.

## Core Score Model

Promptfoo uses four `llm-rubric` assertions, one for each core metric:

| Metric | Weight | Purpose |
|---|---:|---|
| Context Retention | 30 | Preserves requirements, constraints, decisions, and assumptions. |
| Stage Completion | 30 | Produces the expected deliverable for the current stage. |
| Workflow Continuity | 25 | Continues from prior artifacts without drift or contradiction. |
| Reviewability | 15 | Keeps the artifact clear enough for human review. |

The final Long-Horizon score is a weighted average:

```text
score =
  context_retention   * 0.30 +
  stage_completion    * 0.30 +
  workflow_continuity * 0.25 +
  reviewability       * 0.15
```

Promptfoo also runs hard checks, such as required ID preservation and placeholder detection. These checks are reported separately and should be treated as gates or failure reasons.

## Install

Promptfoo requires a modern Node.js runtime. Use Node 20+.

```bash
npm install -g promptfoo
promptfoo --version
```

## Smoke Test

This smoke test uses only the `echo` provider and deterministic assertions. It does not require judge model credentials.

```bash
promptfoo eval \
  -c eval-scripts/promptfoo-long-horizon/promptfooconfig.smoke.yaml \
  --output /tmp/promptfoo-long-horizon-smoke.json
```

## LLM-as-Judge Run

Set a judge model provider and run the main config. Example with an OpenAI-compatible endpoint:

```bash
export OPENAI_API_KEY="<key>"
export OPENAI_BASE_URL="<base-url>"

promptfoo eval \
  -c eval-scripts/promptfoo-long-horizon/promptfooconfig.yaml \
  --output /tmp/promptfoo-long-horizon-results.json
```

To use another judge provider, update the `provider` under the `llm-rubric` assertion in `promptfooconfig.yaml`.

The default config uses:

```yaml
provider: openai:gpt-4.1-mini
```

For an Anthropic-compatible, OpenAI-compatible, or self-hosted judge endpoint, replace that provider with the provider supported by your local Promptfoo setup.

## Summarize Exported Results

```bash
python3 eval-scripts/promptfoo-long-horizon/scripts/summarize_results.py \
  /tmp/promptfoo-long-horizon-results.json \
  --json-out /tmp/promptfoo-long-horizon-summary.json
```

The summary reports:

- Promptfoo test pass rate
- assertion pass rate
- average score per core metric
- weighted Long-Horizon score
- per-case score breakdown

## Notes

- Keep Promptfoo focused on repeatable scoring, hard checks, and reporting.
- Calibrate judge scores with a small human-reviewed sample before scaling the run.
